## 1. DairyComp305 妊娠率の基本概念

### 1.1 定義

**21日妊娠率（21-Day Pregnancy Rate / PR）** は、
21日間のサイクルごとに「繁殖対象牛のうち何頭が妊娠したか」を示す繁殖KPIです。

```
妊娠率 = 妊娠頭数 ÷ 妊娠対象牛頭数 × 100 (%)
```

乳牛の発情周期（平均21日）に合わせたサイクル設計により、
**繁殖の機会損失**（発情見逃し・AI機会損失）まで反映できる点が特徴。
単純な受胎率（Conception Rate）とは異なる指標です。

```
妊娠率 ≈ 授精率（HDR）× 受胎率（CR）
例: 授精率 50% × 受胎率 50% = 妊娠率 25%
```

### 1.2 目標値

| 農場レベル | 21日妊娠率 |
|-----------|-----------|
| 優秀農場 | 25〜35% |
| 平均的農場 | 15〜22% |
| 改善要 | 10〜14% |

**本農場の実績**: 50/203 = **24.6%**（優秀レベル）

## 2. サイクル設定

### 2.1 基本設定

- **1サイクル = 21日**（乳牛の発情周期に対応）
- **標準サイクル数 = 18**（18 × 21 = 378日 ≈ 13ヶ月）
- **基準日**: 農場設定可能（本データでは 2025-03-13 が最初のサイクル開始）

### 2.2 有効サイクルの判定

```
サイクル終了日 + 28日 ≤ 今日（= 妊娠検査が完了しうる期間）
```

最近2サイクルは通常「未確定（妊娠対象=0）」として表示。

**Excelで確認**:
- 2026.2.12-3.5 終了: 3/5 + 28 = 4/2 > 今日(3/26) → 妊娠対象=0 ✓
- 2026.1.22-2.12 終了: 2/12 + 28 = 3/12 ≤ 今日(3/26) → 妊娠対象=14 ✓

## 3. 繁殖対象牛（BR_EL: Eligible Cows / Cows at Risk）

### 3.1 基本条件（4条件全て満たす）

| 条件 | 詳細 |
|------|------|
| **DIM ≥ VWP** | 分娩後の自発的待機期間を超過（判定時点: サイクル開始日） |
| **オープン牛** | 現在妊娠確認されていない |
| **非DNB** | 繁殖停止（Do Not Breed）指定でない |
| **在籍中** | サイクル開始時点で農場にいる |

### 3.2 VWP（Voluntary Waiting Period）

- **DairyComp305デフォルト**: 50日
- **本農場の設定**: **42日**（Excelから逆算・以下参照）
- **FALCON設定**: デフォルト60日（settings_manager.pyで変更可能）

**VWP=42の根拠（Excelから逆算）**:

| サイクル開始日 | 牛ID | DIM@サイクル開始 | 前サイクルDIM | VWP整合 |
|--------------|------|-----------------|-------------|---------|
| 02/12 | 1302 | **42** | 21 | 42≥42✓, 21<42✓ |
| 02/12 | 1430 | 50 | 29 | 50≥42✓, 29<42✓ |
| 01/22 | 1490 | 45 | 24 | 45≥42✓, 24<42✓ |
| 03/05 | 920 | 44 | 23 | 44≥42✓, 23<42✓ |

最小DIM@初登場 = 42 → VWP = 42日（農場個別設定）

### 3.3 在籍カットオフ（サイクル途中の入退場）

DairyComp305の一般的な実装:
- **11日以上在籍** → 1頭としてカウント
- **10日以下在籍** → 除外（0頭）

**Excelで確認できたケース**:
- 牛1343: 10/09-10/30サイクルで在籍12日（10/09〜10/21死亡）→ 繁殖対象に算入 ✓（12 ≥ 11）
- 牛1230: 01/01-01/22サイクルで在籍12日（01/01〜01/13売却）→ 繁殖対象に算入 ✓（12 ≥ 11）
- 10日以下のケースは本データに含まれず、カットオフ値の下限確認は不可

### 3.4 除外されるケース

| 除外条件 | タイミング |
|----------|------------|
| 妊娠確認済み（PDP/PAGP） | 確認日以降のサイクルから除外 |
| DNB（STOPR=204） | 設定日以降のサイクルから除外 |
| 売却（SOLD=205） | 退場日以降のサイクルから除外 |
| 死亡（DEAD=206） | 退場日以降のサイクルから除外 |
| DIM < VWP | 当サイクルは除外 |

### 3.5 VWPカラムの意味

サイクルシートの「VWP」列 = **その牛が初めて繁殖対象に入ったサイクルの開始日**（例: VWP=02/12 → 2/12サイクルが初回）

## 4. 妊娠対象牛（Cows at Risk for PR Calculation）

### 4.1 定義（最重要）

```
妊娠対象牛 = 繁殖対象牛 - 「AIを実施したが結果がS=未確定の牛」
```

繁殖対象牛 ≠ 妊娠対象牛 である点がDairyComp305の重要な設計。
妊娠結果が**確認できていない牛**は分母から外すことで、
未完了データによる誤った妊娠率を防ぐ。

### 4.2 S値（AI結果コード）の定義と処理

| コード | 意味 | 妊娠対象に算入 | 妊娠数にカウント | 損耗 |
|--------|------|:---:|:---:|:---:|
| `P` | Pregnant（妊娠確認） | ✓ | ✓ | — |
| `O` | Open（空胎確認・チェック陰性） | ✓ | — | — |
| `A` | Abort（受胎後流産） | ✓ | ✓ | ✓ |
| `R` | Repeat（同サイクル内再授精） | ✓ | — | — |
| `-(空白)` + **Date有** | AI実施・結果未確定 | **除外** | — | — |
| `-(空白)` + **Date無** | 未授精（当サイクル） | ✓（非妊娠扱い） | — | — |

**S=A（受胎後流産）について**:
- 受胎時点では妊娠 → 妊娠数にカウント（妊娠率に貢献）
- 「損耗（Loss）」カラムに別途記録（管理指標）

### 4.3 計算式のまとめ

```python
# サイクルの妊娠率計算
def calc_pr(cycle_cows):
    if not is_cycle_calculable(cycle):  # cycle_end + 28日 > 今日
        return 0, 0, 0

    preg_eligible = sum(
        1 for cow in cycle_cows
        if not (cow.had_ai_this_cycle and cow.s_result == '-')
    )
    preg_count = sum(
        1 for cow in cycle_cows
        if cow.s_result in ('P', 'A')
    )
    loss_count = sum(1 for cow in cycle_cows if cow.s_result == 'A')
    pr = round(preg_count / preg_eligible * 100) if preg_eligible > 0 else 0

    return preg_count, preg_eligible, pr, loss_count
```

## 6. 授精カウントのルール

### 6.1 授精のサイクルへの割り当て

```
サイクル開始日 ≤ 授精日 ≤ サイクル終了日 → そのサイクルにカウント
```

### 6.2 受胎の帰属先

DairyComp305標準: **授精を実施したサイクルに遡及割り当て**
（妊娠確認日ではなく、AIを行った日のサイクルにカウント）

Excelで確認: ConcカラムはAI日と一致するケースが多く、サイクル内帰属ルールに整合。

### 6.3 Days（DIM）カラムの注意点

サイクルシートの「Days」列:
- AI実施牛: **最終AIイベント日のDIM**（サイクル開始日ではない場合がある）
- AI未実施牛: **サイクル開始日のDIM**

VWP判定・繁殖対象判定には「サイクル開始日のDIM」（= 分娩日からの正確な日数）を使うこと。

```python
def count_preg_target(self, cycle_cows, cycle, all_events):
    """
    妊娠対象牛数 = サイクル内AI実施+結果確認済み + AI未実施の全牛
    除外: AI実施したが結果(PDP/PDN/PAGP/PAGN)が未記録の牛
    """
    count = 0
    for cow in cycle_cows:
        ai_in_cycle = self._get_ai_in_cycle(cow, cycle, all_events)
        if not ai_in_cycle:
            count += 1  # AI未実施 → 非妊娠として算入
            continue
        result = self._get_result_for_ai(cow, ai_in_cycle, all_events)
        if result in ('P', 'O', 'A', 'R'):
            count += 1  # 確認済みのみ算入
    return count

def count_preg_positive(self, cycle_cows, cycle, all_events):
    """
    妊娠数 = S=P または S=A の牛（受胎後流産も妊娠にカウント）
    """
    return sum(
        1 for cow in cycle_cows
        if self._get_result_for_ai(cow, ..., all_events) in ('P', 'A')
    )
```

## 9. 実装サンプルコード（全ロジック統合）

```python
from datetime import timedelta

CYCLE_DAYS = 21
N_CYCLES = 18
RESULT_WAIT_DAYS = 28  # 妊娠確認に必要な日数

def generate_cycles(base_date):
    """基準日から18サイクルを生成"""
    return [
        (base_date + timedelta(days=i * CYCLE_DAYS),
         base_date + timedelta(days=(i + 1) * CYCLE_DAYS - 1))
        for i in range(N_CYCLES)
    ]

def is_eligible(cow, cycle_start, vwp=50):
    """繁殖対象牛かどうかを判定"""
    dim = (cycle_start - cow.last_calving_date).days
    return (
        dim >= vwp             # VWP超過
        and not cow.is_pregnant
        and not cow.is_dnb
        and cow.is_present(cycle_start)
    )

def calc_cycle_pr(cycle_start, cycle_end, today, eligible_cows, ai_results):
    """
    Args:
        ai_results: {cow_id: {'had_ai': bool, 's': 'P'/'O'/'A'/'R'/'-'}}
    """
    if (today - cycle_end).days < RESULT_WAIT_DAYS:
        return {'preg_eligible': 0, 'preg': 0, 'pr': 0, 'loss': 0}

    preg_eligible = 0
    preg = 0
    loss = 0

    for cow in eligible_cows:
        result = ai_results.get(cow.id, {})
        had_ai = result.get('had_ai', False)
        s = result.get('s', '-')

        if had_ai and s == '-':
            continue  # AI実施+未確定 → 妊娠対象から除外
        preg_eligible += 1

        if s in ('P', 'A'):
            preg += 1
        if s == 'A':
            loss += 1

    pr = round(preg / preg_eligible * 100) if preg_eligible > 0 else 0
    return {'preg_eligible': preg_eligible, 'preg': preg, 'pr': pr, 'loss': loss}
```
