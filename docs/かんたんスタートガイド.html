農場ファイルの作成

最初の一歩。農場データの「入れ物」を作ります。

FALCONを起動する

restart_falcon.bat をダブルクリック（またはデスクトップのFALCONアイコン）して起動します。

画面イメージ①：起動直後の農場選択画面

新規農場を作成する（3ステップ）

「新規農場を作成」ボタンをクリックすると、ウィザードが起動します。

  • 1
    農場名を入力
    農場名はフォルダ名になります（日本語OK・スペース不可推奨）。 入力後「次へ」をクリック。
    データ保存先：C:\FARMS\{農場名}\farm.db
  • 2
    保存場所の確認
    農場フォルダの保存先が表示されます。通常はそのまま「次へ」で進みます。
  • 3
    個体登録テンプレートの出力（任意）
    多頭数を一括登録したい場合はExcelテンプレートをここで出力できます。 急がなければ「スキップして完了」でOKです。
完了！農場リストに新しい農場が表示されたら作成完了です。農場名をダブルクリックして開いてください。

個体マスターの登録

牛を登録することで、すべての管理が始まります。

農場を開くと、左側にサイドメニュー、右側にメイン表示エリアが表示されます。

画面イメージ②：メイン画面の構成

方法① 1頭ずつ登録（導入）

新しい個体を1頭ずつ登録するときは、サイドメニュー「牛の管理 → 導入」を開きます。
個体カードに「＋新規登録」のような操作はありません。個体カードは既存の牛の一覧・閲覧やイベント入力に使います。

サイドメニュー › 牛の管理 › 導入
入力項目説明
導入日・登録日農場に入った日、繁殖指標の起点日（登録日が空欄のときは導入日に合わせます）
JPN10（10桁）個体識別耳標の番号。先に入力します。
ID（自動）JPN10を入力すると、正式10桁のうち拡大4桁に相当する部分から農場内ID（4桁）が自動設定されます。
TAG（タグ）任意。農場独自の管理番号（耳番など）を付けたいときに使います。一覧・個体カードのタグと同じ項目です。
品種・生年月日わかれば入力
産次（lact）0＝未経産、1以上＝経産牛。繁殖検診・妊娠率に直結する重要項目。
分娩月日・最終授精日・授精回数・受胎の有無・PEN などわかる範囲で入力すると、導入直後から繁殖管理が正しく動きます
重要：産次（lact）を正確に妊娠率（21日PR）の計算は経産牛（産次≥1）のみ対象です。未経産（産次＝0）は対象外になります。必ず正確に入力してください。

画面イメージ③：導入入力画面

購入・移動で来た牛も同じ外部から入った牛の登録も、この「導入」画面で行います。保存すると導入イベントが記録されます。
過去情報を入力するほど精度が上がります「分娩月日」「最終授精日」などを入力しておくと、導入後すぐに繁殖検診の対象牛として正しく認識されます。わかる範囲で入力しましょう。

イベントの入力

分娩・授精・妊娠鑑定など、牛に起きた出来事を記録します。

イベントとは？牛に起きた出来事（分娩・授精・妊娠鑑定など）を日付とともに記録したものです。FALCONはすべての情報管理、分析をイベントをもとに動かしています。

イベントの入力方法

  • A
    個体カードを開く → 対象牛を選択 → 「イベント入力」ボタン
  • B
    サイドメニュー「牛の管理 → イベント入力」から直接入力

入力項目選択肢
分娩難易度1:自然分娩 / 2:介助 / 3:難産 / 4:獣医師による難産 / 5:帝王切開

💉 人工授精（AI）イベント番号：200

画面イメージ⑤：AI入力

入力項目説明
授精師「農場設定 → 授精設定」で登録したコードから選択
授精種類自然発情 / CIDR / OvSynch 等（種類コードは英字。事前に「授精設定」で登録）

✅ 妊娠鑑定プラス（PDP）イベント番号：303

直近のAIが受胎した場合に使います（最も一般的な妊娠鑑定プラス）。

画面イメージ⑥：妊娠鑑定プラス（PDP）

PDP入力後、繁殖コードは「5：妊娠中」に自動変更されます。分娩予定日もAI日から280日として自動計算されます。
✅ 妊娠鑑定プラス（直近以外）イベント番号：304

直近のAIではなく、それより前のAIが受胎していた場合に使います。
例：「ひとつ前のAIで実は妊娠していた」

画面イメージ⑦：妊娠鑑定プラス（直近以外）

「どのAIが受胎したか」を正しく選ぶことが重要です。分娩間隔・妊娠率の計算に影響します。
✗ 妊娠鑑定マイナス（PDN）イベント番号：302

画面イメージ⑧：妊娠鑑定マイナス（PDN）

PDN入力後、繁殖コードは「4：空胎」に戻ります。次回の繁殖検診では「繁殖検査２」対象として自動リストアップされます。
🧪 PAGプラス（PAGP）/ PAGマイナス（PAGN）イベント番号：307 / 306

PAG（妊娠関連タンパク質）血液検査の結果を記録します。

イベント入力項目
PAGプラス（307）性別（♀/♂、わかれば）、双子チェック
PAGマイナス（306）なし（日付のみ）
PAGPもPDPと同様に「妊娠中」として扱われ、妊娠率（21日PR）の分子にカウントされます。
📋 繁殖停止・売却・死亡
イベント効果
繁殖停止 STOPR（204） 繁殖コード「1：繁殖停止」となり、繁殖検診対象から外れる。AIを入力するかSTOPRを取り消すと解除。
売却 SOLD（205） 牛群から除籍。個体カードが赤く表示される。繁殖検診・妊娠率の計算対象外。
死亡・廃用 DEAD（206） 同上。

繁殖検診をやってみよう！

獣医師の方へ。繁殖検診の流れをステップで解説します。

サイドメニュー › 牛の管理 › 繁殖検診

画面イメージ⑪：繁殖検診フロー画面

📄 ステップ 1：繁殖検診表で対象牛を確認する

「繁殖検診表」→「開く」をクリックします。設定した日数閾値に基づき、今日の検診対象牛が一覧で表示されます。

画面イメージ⑫：繁殖検診表（一覧）

列見方
DIM分娩後日数（自動計算）
繁殖コード現在の繁殖状態（分娩後・空胎・授精後・妊娠中 等）
検診区分今日何をすべきか（フレッシュ・繁殖検査１・妊娠鑑定 等）
一覧は印刷プレビューから印刷できます。空白Excel検診表の出力や、検診表のExcel一括吸い込みはありません。検診の記録は繁殖検診入力で行います。
※ 同じフロー画面の「AI/ET」等の別カードにはテンプレ／吸い込みがありますが、検診表用ではありません。
✏️ ステップ 2：繁殖検診入力を行う

「繁殖検診入力」→「開く」→ 検診日を入力して「抽出」ボタンを押します。対象牛が一覧表示されます。

画面イメージ⑬：繁殖検診入力画面

「処置/結果」コードの使い分け（重要！）

コード意味使うタイミング
2フレッシュチェックフレッシュ区分の牛を検診
3繁殖検査実施繁殖検査１・２の牛を検診
4妊娠鑑定マイナス（PDN）妊娠していなかった
5妊娠鑑定プラス（直近AI）直近のAIで妊娠確認
6妊娠鑑定プラス（直近以外）以前のAIで妊娠確認
7PAGマイナスPAG検査で陰性
8PAGプラスPAG検査で陽性（妊娠確認）
コード5・6・8を選ぶと、その牛は即座に「妊娠中」ステータスに変更されます。入力ミスのないよう確認してから「全件保存」をクリックしてください。
📊 ステップ 3：繁殖検診レポートで結果を確認する

「レポート出力」→「開く」→ ダイアログで検診日を指定し、「繁殖レポート」にチェックを入れてOKを押します。ブラウザにHTMLレポートが開きます。「集計」ボタンはありません。

画面イメージ⑭：繁殖検診レポート

指標見方・目標
妊娠率（21日PR）21日サイクルごとに妊娠した割合。目標20%以上。
受胎率AIした牛のうち妊娠した割合。目標40%以上。
授精実施率繁殖対象牛のうちAIされた割合。目標60%以上。
種類別受胎率プログラム別の効果比較。

獣医師ポイント：妊娠率が低い時の原因分析

妊娠率（21日PR）＝ 受胎率 × 授精実施率

▸ 授精実施率が低い → 発情発見・タイミング同期の問題
▸ 受胎率が低い → 授精技術・牛の健康状態・栄養・熱ストレスの問題

レポートで「どちら」が引き下げているかを見極めることが改善への第一歩です。
