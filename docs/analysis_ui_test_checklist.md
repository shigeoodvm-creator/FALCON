# AnalysisUI 動作確認チェックリスト

## 1. 分析UIの起動確認

### 1.1 メイン画面からの起動
- [ ] メイン画面の「分析」ボタンをクリック
- [ ] AnalysisUIが正しく表示される
- [ ] 左側：分析種別選択、期間指定、コマンド入力欄、参照パネルが表示される
- [ ] 右側：結果表示エリア（表/グラフ/イベント一覧タブ）が表示される

### 1.2 既存機能への影響
- [ ] 「個体カード」ボタンが正常に動作する
- [ ] 「繁殖検診」ボタンが正常に動作する
- [ ] 「イベント入力」ボタンが正常に動作する
- [ ] 「データ出力」ボタンが正常に動作する
- [ ] 画面遷移が正常に動作する

## 2. 基本動線の確認（E2E）

### 2.1 右パネルからの入力
- [ ] 項目タブで項目をクリック → カーソル位置に挿入される
- [ ] 項目タブで項目をダブルクリック → 適切な行に挿入/置換される
- [ ] イベントタブでイベントをクリック → カーソル位置に挿入される
- [ ] イベントタブでイベントをダブルクリック → 適切な行に挿入/置換される
- [ ] DIM区分をクリック/ダブルクリック → 正しく挿入される
- [ ] グラフ種類をクリック/ダブルクリック → 正しく挿入される

### 2.2 ガイド表示
- [ ] 分析種別を変更すると、ガイドが更新される
- [ ] 行に入力があると、その行のガイドが消える
- [ ] 行を空にすると、ガイドが再表示される
- [ ] ガイドが薄字で表示される
- [ ] ガイドをクリックしても入力できる（透過）

### 2.3 コマンド実行 → 結果表示
- [ ] コマンドを入力して「実行」ボタンをクリック
- [ ] 結果が表ビューに表示される
- [ ] エラーがある場合はエラー表示エリアに表示される
- [ ] 警告がある場合は警告表示エリアに表示される

## 3. 各分析種別の最低限テスト

### 3.1 リスト
- [ ] 1行目に項目を入力（例: ID、分娩後日数、産次）
- [ ] 2行目以降に条件を入力（例: 2産以上、DIM<150）
- [ ] 実行すると結果が表に表示される
- [ ] 列名が表示名（日本語）で表示される

### 3.2 集計
- [ ] 1行目に集計項目を入力（例: 授精回数）
- [ ] 2行目にGROUP BY基準を入力（例: 産次）
- [ ] 実行すると結果が表に表示される
- [ ] GROUP BYが正しく機能しているか

### 3.3 イベント集計
- [ ] 1行目にイベントを入力（例: 授精、分娩）
- [ ] 2行目にGROUP BY基準を入力（例: 月）
- [ ] 実行すると結果が表に表示される
- [ ] イベントグルーピング（授精=AI+ET）が正しく機能しているか

### 3.4 グラフ
- [ ] 1行目に縦軸項目を入力
- [ ] 2行目に横軸項目を入力
- [ ] 3行目にグラフ種類を入力（例: 折れ線）
- [ ] 実行するとグラフが表示される（またはエラーメッセージ）

### 3.5 繁殖分析
- [ ] 1行目に繁殖指標を入力（例: 受胎率）
- [ ] 実行すると結果が表示される（またはエラーメッセージ）

## 4. エラーハンドリング

### 4.1 未解決の項目名/イベント名
- [ ] 存在しない項目名を入力 → 分かりやすいエラー/警告が表示される
- [ ] 存在しないイベント名を入力 → 分かりやすいエラー/警告が表示される
- [ ] エラー時にUIが壊れず、再入力できる

### 4.2 不正な条件
- [ ] 不正な条件式を入力 → エラーが表示される
- [ ] エラー後に修正して再実行できる
