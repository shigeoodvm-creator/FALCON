# 繁殖分析（DC305互換）実装完了報告

## 実装日
2024年

## 実装内容

### 1. 新規モジュール作成
- `app/modules/reproduction_analysis.py` - 繁殖分析ロジック（DC305互換）

### 2. 実装機能

#### 2.1 21日サイクルの生成
- デフォルトで起算日から18サイクル遡る
- 期間指定がある場合は、終了日を起算点として開始日まで21日区切りでサイクルを生成
- `generate_cycles()`メソッドで実装

#### 2.2 VWP（Voluntary Waiting Period）対応
- デフォルトVWP: 50日
- 農場設定からVWPを取得（将来的に「農場繁殖指標設定」から取得予定）
- 現時点ではデフォルト値（50日）を使用

#### 2.3 繁殖対象個体（Br El）の判定
- `is_breeding_eligible()`メソッドで実装
- 以下の条件を満たす個体を対象：
  1. VWPを超えている
  2. 空胎（未授精、または不受胎確定）
  3. 繁殖停止がまだ下されていない
  4. 繁殖停止がサイクル途中で下された
  5. 除籍イベントがサイクル途中で行われた
- 以下の個体は除外：
  1. 繁殖停止がサイクル開始前
  2. サイクル開始前の授精で妊娠成立
  3. VWP内
  4. サイクル開始前に除籍
  5. 流産イベントによる空胎個体

#### 2.4 授精数（Bred）の計算
- `count_bred()`メソッドで実装
- AIイベント + ETイベント数をカウント
- 連続授精扱い（Rフラグ、outcome='R'）のイベントは除外

#### 2.5 妊娠数（Preg）の計算
- `count_preg()`メソッドで実装
- サイクル内で受胎が確定した授精数をカウント
- 直近2サイクルは妊娠未確定のため、妊娠数=0とする（`exclude_recent`フラグ）

#### 2.6 指標計算
- 発情発見率（HDR） = 授精数 / 繁殖対象個体数 × 100
- 妊娠率（PR） = 妊娠数 / 繁殖対象個体数 × 100
- 平均値は各サイクルの単純平均

#### 2.7 DC305互換テーブル出力
- `_execute_repro()`メソッドで実装
- 各サイクル行 + Total/Average行を出力
- カラム: cycle, start_date, end_date, br_el, bred, preg, hdr, pr

### 3. ExecutorV2への統合

### 4. 実装ファイル

## 使用方法

### AnalysisUIから実行
1. 分析種別を「繁殖分析」に選択
2. 期間を指定（オプション）
3. 実行ボタンをクリック
4. DC305互換のテーブルが表示される

### 出力形式
```
cycle          | start_date | end_date   | br_el | bred | preg | hdr   | pr
Cycle 1        | 2024-01-01 | 2024-01-21 | 10    | 5    | 3    | 50.0  | 30.0
Cycle 2        | 2024-01-22 | 2024-02-11 | 12    | 6    | 4    | 50.0  | 33.3
...
Total/Average  |            |            | 220   | 110  | 66   | 50.0  | 30.0
```

## 注意事項

### VWP設定
- 現時点ではデフォルト値（50日）を使用
- 将来的に「農場設定＞農場繁殖指標設定」から取得する予定
- 設定ファイル: `farm_settings.json` の `reproduction.vwp` キー

### 直近2サイクルの扱い
- 直近2サイクルは妊娠未確定のため、妊娠数=0として計算
- 発情発見率（HDR）は計算されるが、妊娠率（PR）は0%となる

### 繁殖対象個体の判定
- サイクル開始時点の状態を正確に計算するため、`_calculate_state_at_date()`メソッドを使用
- サイクル開始時点より前のイベントのみで状態を計算
