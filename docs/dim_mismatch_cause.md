# 個体カード「DIM」とイベント履歴のDIMが一致しない原因

## 現象

- **基本情報**: 分娩月日 2026-03-02、DIM 11日
- **イベント履歴**: 直近の分娩は 2025-08-27（DIM 0）、以降のイベントはすべて 2025-08-27 を基準にした DIM（例: 2026-02-09 で DIM 166）

→ 同じ個体なのに「今の DIM」と「履歴に表示される DIM」の基準が違って見える。

## 原因（データの二重管理）

| 表示場所 | 参照元 | 計算方法 |
|----------|--------|----------|
| 基本情報の「分娩月日」 | **cow テーブルの `clvd`** | そのまま表示 |
| 基本情報の「DIM」 | **cow テーブルの `clvd`** | `formula_engine`: 今日 − cow.clvd |
| イベント履歴の各 DIM | **イベントテーブルの分娩イベントのみ** | そのイベント日以前の「直近の分娩**イベント**」からの日数 |

通常は RuleEngine が分娩イベント(202)を適用するたびに `cow.clvd` を更新するため、**cow.clvd とイベント上の分娩日は一致している**前提で動いています。

不一致になる典型的なパターンは次のとおりです。

1. **cow の分娩月日だけが更新されている**
   - 個体編集で「分娩月日」を 2026-03-02 に手動変更した
   - またはインポート・一括更新で cow.clvd だけ 2026-03-02 に設定した  
   → 対応する **分娩イベント（2026-03-02）がイベントテーブルにない**

2. **分娩イベントが未登録**
   - 実際には 2026-03-02 に分娩したが、イベント入力で分娩(202)を登録していない  
   → cow.clvd は何らかの経路で 2026-03-02 になっているが、履歴には 2025-08-27 が最後の分娩

3. **RuleEngine の再計算が行われていない**
   - イベントを追加・削除したあと、cow の状態（lact, clvd, rc）を再計算していない  
   → 本来なら「イベントから計算した clvd」と cow.clvd は一致するはずが、古い値のまま

## 対処方法

### データを正す（推奨）

- **2026-03-02 の分娩が正しい場合**  
  → イベント入力で「分娩」イベントを 2026-03-02 で追加する。  
  追加後に RuleEngine で cow を再計算すれば、cow.clvd とイベントが一致し、基本情報・履歴の DIM も同じ基準になる。

- **2026-03-02 が誤りで、直近分娩が 2025-08-27 のままの場合**  
  → 個体編集で「分娩月日」を 2025-08-27 に戻すか、RuleEngine の「イベントから牛の状態を再計算」を実行して cow.clvd をイベントに合わせる。

### 表示側の対応（実装済み）

イベント履歴の DIM 計算で、**イベントに分娩が無い場合や、cow.clvd の方が新しい場合**に、cow の分娩月日（clvd）を補助的に参照するようにしています。  
これにより「cow.clvd だけ更新されている」状態でも、その日付以降のイベントについては基本情報の DIM と整合した基準で表示されます。  
ただし、**根本的には「分娩はイベントで登録し、cow.clvd は RuleEngine で同期する」運用にすると、今後も不一致を防げます。**

## 参照コード

- 基本情報の DIM: `app/modules/formula_engine.py` の `_calculate_dim(clvd)`（`cow.get('clvd')` を参照）
- 基本情報の分娩月日: `app/ui/cow_card.py` の `_display_basic_info` で `cow.get('clvd', '')`
- イベント履歴の DIM: `app/ui/cow_card.py` の `_calculate_dim_at_event_date`（分娩**イベント**から「その日以前の最新分娩日」を算出、必要に応じて cow.clvd をフォールバックに使用）
