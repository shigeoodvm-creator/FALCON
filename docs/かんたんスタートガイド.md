# FALCON かんたんスタートガイド

> **はじめて使う方へ** ── このガイドを読めば、今日から繁殖検診が始められます。

---

## はじめに

**FALCON**（Farm Analysis for Lactating Cattle Optimization – Nippon Model）は、  
酪農場の繁殖管理を強力にサポートするデスクトップアプリです。

このガイドでは「農場を作る→個体を登録する→イベントを入力する→繁殖検診を行う」  
という一連の流れを、順を追ってやさしく解説します。

> **対象ユーザー:** 繁殖検診を担当する獣医師、農場スタッフ  
> **対象OS:** Windows 10 / 11

## 1. 農場ファイルの作成

### FALCONを起動する

`restart_falcon.bat` をダブルクリック（またはデスクトップのFALCONアイコン）して起動します。

```
┌─────────────────────────────────────────┐
│  🐄  FALCON                             │
│  農場を選択してください                   │
│                                         │
│  （農場リスト）                          │
│                                         │
│  [ 農場を開く ]  [ 新規農場を作成 ]      │
└─────────────────────────────────────────┘
```
> **【画面イメージ①】** 起動直後の農場選択画面

---

### 新規農場を作成する（3ステップ）

「**新規農場を作成**」ボタンをクリックすると、3ステップのウィザードが開きます。

---

#### ステップ 1 ── 農場名を入力

```
┌─────────────────────────────────────────┐
│  🏠 農場作成  ステップ1/3               │
│                                         │
│  農場名:  [北海道〇〇牧場          ]    │
│                                         │
│           [ 次へ → ]                    │
└─────────────────────────────────────────┘
```
> **【画面イメージ②】** 農場名入力

- 農場名はフォルダ名になります（日本語OK・スペース不可推奨）
- データは `C:\FARMS\{農場名}\farm.db` に保存されます

---

#### ステップ 2 ── 保存場所の確認

農場フォルダの保存先が表示されます。通常はそのまま「**次へ**」で進みます。

---

#### ステップ 3 ── 個体登録テンプレートの出力（任意）

個体を一括登録したい場合は、Excelテンプレートをここで出力できます。  
→ 「**テンプレートを出力してスキップ**」または「**スキップして完了**」

> **ポイント:** テンプレートはあとでも出力できるので、急がなくてOKです。

---

**これで農場ファイルの作成は完了！** 農場リストに新しい農場が追加されます。  
農場名をダブルクリックして開いてください。

```

│    導入          │                                          │

│                  │                                          │
│  ▼ 帳票・分析    │                                          │
│  ▼ データ管理    │                                          │
│  ▼ 個体チェック  │                                          │
│  ▼ 設定・管理    │                                          │
└──────────────────┴──────────────────────────────────────────┘
```
> **【画面イメージ③】** メイン画面の構成

---

### 1頭ずつ登録する（導入）

新しい個体を1頭ずつ登録するときは、サイドメニュー「**牛の管理 → 導入**」を開きます。  
※ **個体カードに「＋新規登録」のような操作はありません。** 個体カードは既存の牛の一覧・閲覧やイベント入力に使います。

| 入力項目 | 説明 |
|----------|------|
| 導入日・登録日 | 農場に入った日、繁殖指標の起点日（登録日が空欄のときは導入日に合わせます） |
| JPN10（10桁） | 個体識別耳標の番号。先に入力します。 |
| ID（自動） | JPN10を入力すると、正式10桁のうち**拡大4桁**に相当する部分から**農場内ID（4桁）**が自動設定されます。 |
| TAG（タグ） | 任意。**農場独自の管理番号**（耳番など）を付けたいときに使います。一覧・個体カードのタグと同じ項目です。 |
| 品種・生年月日 | わかれば入力 |
| **産次（lact）** | **0＝未経産、1以上＝経産牛。** 繁殖検診・妊娠率に直結します。 |
| 分娩月日・最終授精日・授精回数・受胎の有無・PEN など | わかる範囲で入力すると、導入直後から繁殖管理が正しく動きます |

> **ポイント:** 「**産次**」は繁殖検診・妊娠率に直結するため、必ず正確に入力してください。  
> 妊娠率（21日PR）の計算は**経産牛（産次≥1）のみ**対象です。未経産（産次＝0）は対象外です。

購入・移動などで外部から牛が来た場合も、同じ「**導入**」画面で登録します。保存すると導入イベントが記録されます。  
分娩月日・最終授精日など、わかる範囲を入力しておくと、導入直後から繁殖検診で正しく扱われやすくなります。

---

### 個体を一括登録する（Excelインポート）

多頭数を登録するときは、Excelテンプレートを使った一括登録が便利です。

**手順：**
1. サイドメニュー「**データ管理 → データ吸い込み**」を開く
2. 「**個体登録（Excelインポート）**」を選択
3. テンプレートを出力し、Excelに個体情報を入力
4. 入力済みファイルを選択して「**取り込む**」

## 3. イベントの入力

### イベントとは？

牛に起きた出来事（分娩・授精・妊娠鑑定など）を日付とともに記録したものが「**イベント**」です。  
FALCONはすべての情報管理、分析をイベントをもとに動かしています。

**イベントの入力方法：**

1. 個体カードを開く（サイドメニュー「**牛の管理 → 個体カード**」）
2. 対象牛を選択
3. 「**イベント入力**」ボタンをクリック

または

- サイドメニュー「**牛の管理 → イベント入力**」から直接開く

---

```
│                                          │
│  日付:        [ 2026-04-12 ]             │
│  分娩難易度:  [ 2：介助      ▼ ]         │
│  子牛頭数:    [ 1 ]  □ 双子             │
│  子牛性別:    [ ♀ ▼ ]  [ ♀ ▼ ]（双子時）│
│                                          │
│            [ 保存 ]                      │
└──────────────────────────────────────────┘
```
> **【画面イメージ⑤】** 分娩イベント入力

| 入力項目 | 選択肢 | 説明 |
|----------|--------|------|
| 分娩難易度 | 1:自然分娩 / 2:介助 / 3:難産 / 4:獣医師難産 / 5:帝王切開 | 難易度を選ぶ |
| 双子チェック | チェックON/OFF | 双子の場合はON |
| 子牛性別 | ♀ / ♂ | 子牛ごとに選択 |

> **ポイント:** 分娩を入力すると、**産次（lact）**が自動的に更新され、  
> DIM（分娩後日数）のカウントが始まります。

```
┌──────────────────────────────────────────┐
│  📅 AI入力                               │
│                                          │
│  日付:        [ 2026-04-12 ]             │
│  授精師:      [ 1:佐藤     ▼ ]           │

│  精液（種雄牛）: [              ]        │
│                                          │
│            [ 保存 ]                      │

```
> **【画面イメージ⑥】** AI入力

| 入力項目 | 説明 |
|----------|------|
| 授精師 | 事前に「授精設定」で登録したコードから選択 |
| 授精種類 | 自然発情 / CIDR / OvSynch 等（**コードは英字**。事前に「授精設定」で登録） |
| 精液（種雄牛） | 省略可 |

#### 妊娠鑑定プラス PDP（303）

直近のAIが受胎した場合に使います（最も一般的な妊娠鑑定プラス）。

```
┌──────────────────────────────────────────┐
│  📅 妊娠鑑定プラス（PDP）                 │
│                                          │
│  日付:   [ 2026-04-12 ]                  │
│  □ 双子                                  │
│  性別:   [ ♀ ▼ ]                         │
│                                          │
│            [ 保存 ]                      │
└──────────────────────────────────────────┘
```
> **【画面イメージ⑦】** 妊娠鑑定プラス（PDP）

| 入力項目 | 説明 |
|----------|------|
| 双子チェック | 双子妊娠の場合はON |
| 性別（♀/♂） | 判明した場合のみ入力。わからなければ空欄でOK |

> **ポイント:** PDP入力後、その牛の繁殖コードは「**5：妊娠中**」に自動変更されます。  
> 分娩予定日もAI日から280日として自動計算されます。

---

#### 妊娠鑑定プラス（直近以外）PDP2（304）

直近のAIではなく、それより前のAIが受胎していた場合に使います。  
例：「ひとつ前のAIで妊娠していた」

```
┌──────────────────────────────────────────┐
│  📅 妊娠鑑定プラス（直近以外）           │
│                                          │
│  日付:   [ 2026-04-12 ]                  │
│                                          │
│  受胎したAI/ETイベントを選択:            │
│  ┌────────────────────────────────┐      │
│  │ 2026-01-10  AI  （42日前）     │ ←選択│
│  │ 2026-02-01  AI  （21日前）     │      │
│  └────────────────────────────────┘      │
│                                          │
│  □ 双子                                  │
│            [ 保存 ]                      │
└──────────────────────────────────────────┘
```
> **【画面イメージ⑧】** 妊娠鑑定プラス（直近以外）

| 入力項目 | 説明 |
|----------|------|
| AI/ETイベント選択 | その牛の過去AI履歴が一覧表示される → 受胎した回を選ぶ |
| 双子チェック | 双子妊娠の場合はON |

> **注意:** 「どのAIが受胎したか」を正しく選ぶことが重要です。  
> 分娩間隔・妊娠率の計算に影響します。

---

#### PAGプラス PAGP（307）

PAG（妊娠関連タンパク質）検査で陽性だった場合。

| 入力項目 | 説明 |
|----------|------|
| 性別（♀/♂） | 判明した場合のみ |
| 双子チェック | 双子の場合はON |

> **ポイント:** PAGPもPDPと同様に「妊娠中」として扱われ、妊娠率の分子にカウントされます。

---

#### 妊娠鑑定マイナス PDN（302）

```
┌──────────────────────────────────────────┐
│  📅 妊娠鑑定マイナス（PDN）              │
│                                          │
│  日付:         [ 2026-04-12 ]            │
│  処置:         [              ]          │
│  子宮所見:     [              ]          │
│  右卵巣所見:   [              ]          │
│  左卵巣所見:   [              ]          │
│  その他:       [              ]          │
│  発情周期Day:  [    ]                    │
│                                          │
│            [ 保存 ]                      │
└──────────────────────────────────────────┘
```

| 入力項目 | 説明 |
|----------|------|
| 処置 | WPG / GnRH / CIDR 等（任意） |
| 子宮・右卵巣・左卵巣所見 | 所見テキスト（任意） |
| 発情周期Day | 発情周期の何日目か（任意） |

> **ポイント:** PDN入力後、繁殖コードは「**4：空胎**」に戻ります。  
> 繁殖検診では次回の「繁殖検査２」対象として自動リストアップされます。

---

#### 繁殖停止 STOPR（204）

繁殖を止める牛に登録します。

> **ポイント:** STOPR入力後は「**1：繁殖停止**」となり、繁殖検診の対象から外れます。  
> 再開するにはSTOPRを取り消すか、新たにAIを入力してください。

---

#### 売却 SOLD（205）・死亡 DEAD（206）

除籍の場合に入力します。

> **ポイント:** 除籍後は牛群から外れ、繁殖検診・妊娠率の計算対象になりません。  
> 個体カードには赤く表示されます。

## 4. 各種設定（繁殖検診の準備）

繁殖検診を正しく動かすには、事前に設定が必要です。  
サイドメニュー「**設定・管理 → 農場設定**」を開いてください。

```
┌──────────────────────────────────────────┐
│  🏠 農場設定                             │
│  設定項目を選択してください              │
│                                          │
│  📄 授精設定     [ 開く ]                │
│  📋 PEN設定      [ 開く ]                │
│  📝 繁殖検診設定  [ 開く ]               │
│  💊 繁殖処置設定  [ 開く ]               │
│  🧪 農場薬品設定  [ 開く ]               │
│  🎯 目標設定     [ 開く ]                │
│                                          │
│            [ 閉じる ]                    │
└──────────────────────────────────────────┘
```
> **【画面イメージ⑨】** 農場設定メニュー

---

### 4-1. 授精設定（最初に必ずやること）

「**授精設定**」を開きます。

#### 授精師の登録

| コード | 授精師名 |
|--------|---------|
| 1 | 佐藤（例） |
| 2 | 山田（例） |

各行に「コード」と「名前」を入力して保存します（授精師のコードは数字でも構いません）。

#### 授精種類の登録

| コード | 授精種類名 |
|--------|-----------|
| A | 自然発情 |
| B | OvSynch |
| C | CIDR |

**授精種類のコードは半角英字のみです（数字は使えません）。** 画面では英大文字で保存されます。授精師コードと混同しないようご注意ください。

> **なぜ必要か？** AIイベント入力時にプルダウンで選べるようになります。  
> 繁殖検診レポートでも「種類別受胎率」として集計されます。

---

### 4-2. PEN設定

群（PEN）のコードと表示名を登録します。

| コード | PEN名 |
|--------|-------|
| 1 | Lactating 1 |
| 2 | Lactating 2 |
| 3 | Dry |
| 4 | Heifer |

> **ポイント:** PENを設定すると、繁殖検診表に「群別」の表示ができるようになります。

---

### 4-3. 繁殖検診設定（繁殖検診の核心）

「**繁殖検診設定**」を開くと、各検診区分のON/OFFと日数閾値が設定できます。

```
┌──────────────────────────────────────────┐
│  繁殖検診設定                            │
│                                          │
│  ── 経産牛 ──                            │
│  ☑ フレッシュチェック   N日: [ 30 ]      │
│  ☑ 繁殖検査１                           │
│  ☑ 繁殖検査２                           │
│  ☑ 妊娠鑑定           N日: [ 30 ]       │
│  ☑ 再妊娠鑑定          N日: [ 60 ]       │
│  □ 任意妊娠鑑定         N日: [    ]      │
│  ☑ 分娩予定超過        N日: [ 14 ]       │
│  □ チェック                              │
│                                          │
│  ── 未経産牛 ──                          │
│  ☑ 育成繁殖１         月齢: [ 12 ]       │
│  ☑ 育成繁殖２                           │
│  ☑ 育成妊娠鑑定        N日: [ 30 ]       │
│  ☑ 育成再妊娠鑑定      N日: [ 60 ]       │
│                                          │
│          [ 保存 ]                        │
└──────────────────────────────────────────┘
```
> **【画面イメージ⑩】** 繁殖検診設定

| 検診区分 | 対象牛の条件 |
|----------|-------------|
| **フレッシュチェック** | 分娩後N日以上経過、まだ未検診 |
| **繁殖検査１** | フレッシュ後、AI/ET歴がない牛 |
| **繁殖検査２** | AI後に妊娠鑑定マイナスがあり、現在オープンの牛 |
| **妊娠鑑定** | 最終授精からN日以上経過 |
| **再妊娠鑑定** | 妊娠中で、授精からN日以上経過 |
| **分娩予定超過** | 分娩予定日からN日以上超過 |
| **育成繁殖１** | 月齢N以上でAI歴なし（未経産牛） |

> **推奨設定（最初はデフォルトのままでOK）:**  
> - フレッシュチェック：30日  
> - 妊娠鑑定：30日  
> - 再妊娠鑑定：60日  
> - 分娩予定超過：14日

---

### 4-4. 繁殖処置設定

繁殖検診で使う「処置プログラム」を定義します。  
「**繁殖処置設定**」を開いてください。

各処置には「コード」「名前」「使用する薬品プロトコル」を設定します。

| 例：処置コード | 処置名 | 使用薬品 |
|---------------|--------|---------|
| 1 | WPG | PG注射 |
| 2 | CIDR | 膣内留置 + GnRH |
| 3 | OvSynch | GnRH → PG → GnRH |

> **ポイント:** ここで定義した処置が、繁殖検診入力画面のプルダウンに表示されます。  
> 農場で実際に使うプログラムのみ登録しておくと入力がスムーズです。

---

### 4-5. 農場薬品設定

繁殖処置で使用する薬品の製品名と単価を登録します。  
「**農場薬品設定**」を開いてください。

| カテゴリ | 例 |
|---------|-----|
| PG製剤 | エストラメート、プロネスト 等（各3種まで） |
| GnRH製剤 | ファーティレリン、コンセラール 等（各3種まで） |
| 膣内留置製剤 | CIDR、PRID 等（各3種まで） |
| その他 | 任意薬品（5種まで） |
| 処置料 | 超音波検査、筋肉注射、膣内留置、薬治 各単価 |

> **ポイント:** 薬品と処置料を登録することで、繁殖検診レポートに  
> 「1頭あたりの薬品コスト」が自動集計されます。

---

### 4-6. 目標設定（農場KPI）

「**目標設定**」を開き、農場の目標値を設定します。

| 項目 | デフォルト | 説明 |
|------|-----------|------|
| 経産牛頭数 | （未設定） | 規模確認用 |
| 受胎率 | 40% | AI1回あたりの受胎率目標 |
| 授精実施率 | 60% | サイクル内に授精できた割合 |
| **妊娠率（21日PR）** | **20%** | **繁殖効率の核心指標** |
| 分娩間隔 | 420日 | 目標分娩間隔 |
| 初産割合（更新率） | 30% | |

> **ポイント:** 目標値はダッシュボードのグラフに「目標ライン」として表示されます。  
> 現状との差を視覚的に確認できます。

### （参考）繁殖検診フローのほかのカードと Excel

**「繁殖検診表」カードには「開く」だけ**です。

## まとめ：最初の1週間でやること

```
Day 1  農場ファイルを作成する
Day 1  個体マスターを登録する（手動 or Excelインポート）

Day 2  農場設定を行う
       ├─ 授精設定（授精師・授精種類を登録）
       ├─ PEN設定（群のコードを登録）
       ├─ 繁殖検診設定（日数閾値を確認）
       ├─ 繁殖処置設定（WPG/CIDR等を登録）
       └─ 農場薬品設定（使用薬品を登録）

Day 3  過去のイベントを入力する
       ├─ 分娩イベント（CALV）
       ├─ AIイベント
       └─ 妊娠鑑定結果（PDP/PDN）

Day 4  繁殖検診表を確認する
Day 5  繁殖検診入力を実施する
Day 6  繁殖検診レポートで結果を確認する
Day 7  ダッシュボードで牛群全体を俯瞰する
```

## 困ったときは

| 症状 | 確認ポイント |
|------|------------|
| 繁殖検診表に牛が出てこない | 設定で該当区分がONになっているか／**産次（lact）**が正しいか |
| 妊娠率が計算されない | 経産牛（lact≥1）が登録されているか／サイクルが成立しているか |
| AIのプルダウンが空 | 農場設定 → 授精設定 で授精師・授精種類を登録する |
| 繁殖コードがおかしい | 個体カードのイベント履歴を確認し、入力ミスを修正する |
